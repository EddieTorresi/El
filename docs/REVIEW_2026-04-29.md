## 2a. Add a guard so this stops happening

Since this is the third truncation, a 10-line **pre-commit hook** ends the cycle:

```bash
# .git/hooks/pre-commit
#!/usr/bin/env bash
set -e
LINES=$(wc -l < index.html)
TAIL=$(tail -c 200 index.html)
if [ "$LINES" -lt 5200 ]; then
  echo "❌ index.html is only $LINES lines (expected 5200+)"; exit 1
fi
if ! echo "$TAIL" | grep -q "</html>"; then
  echo "❌ index.html does not end with </html>"; exit 1
fi
echo "✓ index.html integrity OK ($LINES lines)"
```

Drop it in `.git/hooks/pre-commit`, `chmod +x`, and the broken file can no longer reach a commit. Belt-and-suspenders: a GitHub Action running the same check on push so a Codex commit from a different machine can't sneak past either.

## 3. What's still worth doing (tightened for solo iPhone + Pages)

### 3a. Visible build tag in the app
The `<!-- El fix build: ... -->` comment isn't visible at runtime, so when you reload on the phone you can't tell whether the fresh fix is actually live or whether the SW is serving last week's HTML. A tiny `<div id="build-tag">` rendered into Settings (or a long-press easter egg on the logo) showing the same string fixes that. Pair it with the next item.

### 3b. Bump the SW cache name with the build string
Per `CONTEXT.md` — "service worker handles offline, don't complicate it" — so I'm *not* suggesting stale-while-revalidate. The minimum change that keeps the SW simple:

```js
// sw.js
const BUILD = '2026-04-29-bug-fixes-v2';
const EL_CACHE = 'el-pages-' + BUILD;
```

Bump `BUILD` in the same edit as the `<!-- El fix build -->` comment in `index.html`. Old caches drop on activation (already implemented), so every release self-invalidates without you remembering to type a new version.

### 3c. Backup reminder (this *is* on your planned list)
Export/Import is built — what's missing is the nudge. A 30-day soft prompt on Home: "It's been 30 days since your last export — back up now?" Implementation is one timestamp in `d.settings.lastExportAt`, set inside the existing export handler, checked on `El.init()`.

### 3d. Narrow XSS escaping for AI output only
Solo user means you don't need broad escaping for fields you type yourself, but the Claude/OpenAI flow stuffs model output into `innerHTML`. The chat path already has `escapeHtml` and uses it on confirmation cards — extend it to the title/notes fields rendered inside `[[EL_ACTION]]` confirmation cards and to `cleanReply` before the `<br>` substitution. ~10 lines, removes the only realistic injection vector.

### 3e. Manifest icon entries
`manifest.webmanifest` lists `el-icon.svg` twice with `purpose: any maskable`. iOS ignores SVG icons entirely (your iPhone uses `el-icon-180.png` via `<link rel="apple-touch-icon">`), and Android maskable expects an 80%-safe-zone — a generic SVG with full-bleed content gets cropped. Drop the two SVG entries from `icons[]` (keep the `<link>` in `<head>` for desktop) and add a dedicated maskable PNG with the diamond pulled inward. Five-minute fix, removes the cropped-icon look on Android if you ever sideload there.

### 3f. `netWorthHistory` retention
Currently capped at 90 entries. The Finance Clarity Engine on your roadmap is going to want long-term trend lines. Bump to ~1,100 (≈ 3 years) before you start charting — the array stays tiny in storage and you don't lose history when you actually need it.

### 3g. Tiny in-file dedupe
- The `cats.map(c => `<option value="${c.id}">${c.icon} ${c.name}</option>`).join('')` pattern appears in 3 modal-open paths. Hoist to `El.ui.catOptions()`.
- `defaults()` rebuilds the same six-category array on every call — pull `DEFAULT_CATEGORIES` out as a module-level constant.
- The expected-path `console.warn` / `console.error` calls (3 of them) could be gated behind `El.debug` so production logs are clean.

These are 5-minute changes, not architectural. Safe to do incrementally.
